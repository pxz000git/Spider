　　作者：何宗丞　　微软在美国雷蒙德一年一度的 Build 大会结束不久，在大洋彼岸的五个城市，一场以“Live to code”为主题的 Windows 8  应用开发马拉松活动正在展开。近 4000 名参与活动的开发者将进行 36  个小时不间断开发，朝西雅图微软总部之旅的大奖进行冲击。从开发者群体分布的广泛性以及现场的气氛来看，微软取得了不错的反响。我们所在北京赛区，学生开发者随处可见。在上海赛区甚至现身  13 岁的开发者。　　正如 Horace Dediu 所说，“你来是因为产品，你留下是因为生态系统。”新生的 Windows 8  能否成功，很大程度取决于其应用生态圈的建设和发展。特别是对于无法运行 Windows 应用的 Windows  RT，微软几乎需要重新打造一个生态圈。微软希望通过这种活泼的方式鼓励开发者的积极性，同时以最短的开发周期为应用商店注入新鲜的血液。据统计，Windows 8  应用商店的应用数量尚不足一万。　　一个平台的生态系统由系统本身、用户以及开发者构成，而开发者又决定了最初的市场规模。在打造 Windows 8  应用生态方面，微软面临的问题仍然很多。　　从 Windows 8 起，微软开始打造自己的应用程序商店，事实上这些应用已经被封闭在 Windows 8  应用商店之中。从移动平台的经验上看，好处是显而易见的，比如可以减少恶意软件的风险，统一支持方式，减少开发者的推广成本。但另一方面，对于习惯了开放了几十年的  Windows 平台的开发者，封闭也意味着一种限制。 据此前 Wired 的采访，已经有一些开发者对此怨声载道。　　就这个问题，微软大中华区开发工具及平台事业部经理 Mark Taylor  称，网上商店也好，应用商店也好，没有哪一家公司能够去主导和控制大家的选择，大家的选择完全是基于市场和客户以及消费者自己的选择。而微软方面，会根据算法将评分高和点击量高的应用放在排名靠前的位置。对于开发者，通过自己的各种渠道进行推广也非常重要。　　“我们要确保 Windows 8  这些产品有一个良性循环和互动的过程。当开发人员看到这些应用销售不断攀升的时候，他知道这个市场的潜力是巨大的。”　　但凡一个生态圈建立之初都得面对“鸡”和“蛋”的悖论，不过凭借微软强大的资金和资源优势，构建生态圈的第一步应该不会太难。从开发者的角度去看，当所有人纷纷扎堆  Android 和 iOS 生态圈却撞得头破血流时，Windows 8  或许是一片广阔的蓝海，当然这需要开发者的胆识和魄力，而微软需要给予开发者足够的信心。　　为了最大化地吸引开发者，微软针对中国开发者社区降低了门槛——即便没有国际信用卡进行 Windows 应用商店账户注册，仍然可以通过第三方 CSDN  渠道向 Windows 应用商店提交应用。据此前 Win App Report 发布的数据，中国本地化的 Windows 8 应用已经超过了美国本土。　　分成模式方面，Windows 8 基本沿用业界普遍三七比例，不过当应用下载量达到一定规模的时候，比例会相应变成二比八，不过这项机制仅限于Windows  8及 Windows RT。　　在问及微软如何保持应用生态圈的潜力和健康时，Taylor 告诉爱范儿，微软会从 Google  和苹果吸取教训，取得一个质与量的平衡。具体在应用审核通常基于三个角度，第一是技术测评，二是政策测评，三是质量测评。微软会将主观性的质量变成量化的标准，比如从设计语言、运行速度以及功能特点等方面入手。　　尽管活动只有短短 36 个小时，但“马拉松”的命名显然暗示着微软细水长流的愿景。对于刚刚起步的 Windows  8，应用生态的打造更像是一场漫长的拉力赛，耐力和韧性不仅仅是开发者所需要的。
